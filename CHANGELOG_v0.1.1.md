# Span v0.1.1 Release Notes

## 🎯 项目定位

**Span** = fscan 的互补搭档，不是替代品。

> fscan 告诉你内网有什么，Span 告诉你这些是什么、怎么打、先打哪个。

目标用户：中级渗透工程师、自学网安学生、用 fscan 扫完不知道怎么继续的人。

---

## ✅ v0.1.1 修复内容

### Bug 修复
1. **修复攻击建议输出重复问题**  
   - 之前："条件:" 行出现两次（一次来自命令模板，一次来自攻击解释）  
   - 现在：只输出一次，格式清晰  

2. **改进角色推断**  
   - 新增 "DNS 服务器" 角色规则（端口 53）  
   - 现在显示前 3 个角色候选（按置信度排序），而不是只显示第一名  
   - 示例：某主机同时开放 22 (SSH) 和 53 (DNS)，会显示：  
     ```
     [角色] Linux SSH 服务器 (置信度: 72%)
     [候选] DNS 服务器 (置信度: 65%)
     ```

3. **改进"无开放端口"输出**  
   - 之前：只显示 "[-] 无开放端口"  
   - 现在：增加提示，说明可能仅开放 Web 端口（80/443），建议用 nmap/fscan 全端口扫描

## 📖 使用示例

### 1. 扫描整个网段
```bash
span 192.168.52.0/24 -threads 50
```

### 2. 详细模式（含教学解释）
```bash
span 192.168.52.0/24 -v
```

### 3. 仅显示关键攻击面（Critical 级别）
```bash
span 192.168.52.0/24 -critical-only
```

### 4. 分析 fscan/nmap 输出
```bash
span --analyze fscan_result.txt -v
span --analyze nmap_result.xml -v
```

### 5. 自定义端口和并发
```bash
span 192.168.52.0/24 -ports 22,445,3389 -threads 100 -timeout 5000
```

## 🐛 已知问题

1. **二进制体积较大**（2.4 MB）  
   - 目标：< 1 MB  
   - 原因：Go 静态编译包含运行时  
   - 解决：后续用 UPX 压缩或优化依赖  

2. **角色推断可能不准确**  
   - 基于端口组合推测，可能误报  
   - 解决：增加更多启发式规则（TTL + Banner）  

3. **OS 识别准确度有限**  
   - 仅基于 TTL + 端口组合，无 Banner 指纹库  
   - 解决：后续版本增加 Banner 关键词匹配
